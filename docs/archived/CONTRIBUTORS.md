## How to Contribute

We welcome contributions! Please see [CONTRIBUTING.md](CONTRIBUTING.md) for guidelines.

### Areas Where We Need Help

- **Plugin Development**: Create new data source plugins
- **Documentation**: Improve guides and API reference
- **Testing**: Write integration and unit tests
- **Bug Reports**: Report and help fix issues
- **Feature Requests**: Suggest and implement new features
- **Code Review**: Review pull requests
- **Documentation Translation**: Translate docs into other languages

## Recognition

Contributors are recognized in:
- Release notes
- Project documentation
- GitHub contributors graph

## Becoming a Maintainer

Active contributors who consistently make valuable contributions may be invited to become maintainers. Maintainers have:

- Write access to the repository
- Ability to merge pull requests
- Participation in project decisions
- Access to internal project resources

If you're interested in becoming a maintainer, demonstrate your commitment through:
- Consistent quality contributions
- Helpful code reviews
- Active participation in discussions
- Support for other contributors
